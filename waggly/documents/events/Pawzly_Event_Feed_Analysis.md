## PHASE 2: FUNCTIONAL CATEGORIES & VALUE PROPOSITIONS

### Category 1: SOS & Emergency Assistance
**Description:**  
Urgent, geographically targeted requests for lost/injured animals, emergency funds, or immediate physical help.

**User Flows:**

**Owner Creates SOS Event:**
1. Account ‚Üí New Event
2. Event Type: "SOS/Help" (Urgent flag)
3. Select pet(s) and situation (Lost, Injured, Urgent Funds, Other)
4. Add photos, location, description
5. Set urgency level (Immediate, Within 24h, ASAP)
6. Share contact + visibility settings
7. Post ‚Üí Notification sent to users within 10km radius + followers

**Volunteer Responds:**
1. See SOS in feed/notifications
2. Tap "I Can Help"
3. Comment/message organizer
4. Confirm assistance, exchange details
5. Mark as "Helping" in feed
6. Post update after resolution

**Key Features:**
- Geofencing: notifications within user-defined radius (default 10km)
- Urgency hierarchy: Immediate (red), 24h (orange), Open (yellow)
- QR tag integration: If pet has Waggli QR collar tag, auto-fill pet info
- Verification: Show verified vet clinic, adoption center, or established profile badges
- Auto-escalation: If unresolved after 2h, escalate to nearby shelters/vets
- Comments timeline: Supporters post updates, sightings, confirmations

**Business Value:**
- Drives QR tag adoption (owners want rapid emergency notification)
- Volunteer retention: emotional reward loop
- Data: Emergency patterns (breed, location, season) ‚Üí predictive analytics
- Sponsorship: Pet insurance, emergency vet clinics sponsor SOS feeds in regions

**Engagement Metrics:**
- Target: 50% of SOS posts resolved within 24h
- Stretch: 90% conversion to help offered within 2h
- Community: Each SOS attracts 10-30 engaged helpers per incident

```typescript
  id: string;

    duration?: number; // for view time
    commentText?: string;
    sentiment?: "positive" | "neutral" | "negative"; // AI-inferred
  };

```

### Event Feed Algorithm (Personalization Engine)

**Ranking Signals (Weighted):**
1. **Recency (15%):** Fresher posts prioritized; exponential decay over 7 days
2. **Engagement Rate (20%):** Likes/comments/saves per impression (faster engagement boost higher)
3. **Social Graph Proximity (25%):** Posts from followed accounts, friends, or accounts similar to followed
4. **User Behavior Similarity (15%):** If user engaged with health posts, surface similar health posts
5. **Location Proximity (15%):** Events/SOS near user's home weighted higher
6. **Verification/Trust (10%):** Verified vets, shelters, established accounts boosted; new accounts slightly deprioritized

**Feed Personalization Rules:**
- SOS posts: Always surface within 2h of creation, regardless of other signals
- Health updates: Surface if user has pet of similar breed, age, health condition
- Local events: Boost if within user's preferred activity radius
- Blood donor: Boost if user registered as potential donor
- Diversity: Ensure feed contains mix of all 4 event types (not 100% health updates)

**Algorithm Implementation:**
- Real-time feed: Use PostgreSQL full-text search + Redis caching for hot posts
- Batch personalization: Daily job re-scores all events for each user
- A/B testing: Compare algorithm variants for engagement, retention, time-to-action

### Data Pipeline for Insights

**Event Stream Processing (Kafka/Supabase Realtime):**
1. New event created ‚Üí Emits event creation event
2. User engages ‚Üí Emits engagement event
3. Event resolved ‚Üí Emits resolution event
4. All events streamed to data warehouse (BigQuery/Snowflake)

**Aggregated Analytics:**
```
-- Daily health trends
SELECT 
  DATE(created_at) as date,
  health_topic,
  COUNT(*) as posts,
  AVG(engagement_rate) as avg_engagement,
  AVG(EXTRACT(YEAR FROM AGE(NOW(), pet_birthdate))) as avg_pet_age
FROM waggli_events
WHERE event_type = 'health_update'
GROUP BY date, health_topic;

-- SOS resolution metrics
SELECT 
  event_type,
  urgency_level,
  AVG(EXTRACT(EPOCH FROM (resolved_at - created_at))/3600) as avg_resolution_hours,
  COUNT(*) as total_cases
FROM waggli_events
WHERE event_type = 'sos' AND status = 'resolved'
GROUP BY urgency_level;

-- Blood donor network health
SELECT 
  location,
  blood_type,
  COUNT(DISTINCT donor_id) as registered_donors,
  COUNT(*) as donations_fulfilled,
  AVG(days_between_donations) as avg_donation_interval
FROM blood_donor_events
GROUP BY location, blood_type;
```

**B2B Insights (Anonymized for Clients):**
- **Insurance:** Aggregate health trends by breed, age, location (claim prediction)
- **Veterinary Chains:** Peak injury times, preventive care gaps, seasonal disease patterns
- **Pet Food Companies:** Nutrition topic popularity, product recommendation effectiveness
- **Training Platforms:** Training methodology effectiveness, progression timelines

## PHASE 8: RISK MITIGATION & SAFEGUARDS

### Content Moderation
| Risk | Mitigation |
|------|-----------|
| Medical misinformation | AI pre-screening + vet sign-off for health claims; watermark: "Not verified medical advice" |
| Safety concerns (SOS) | Verify profiles; flag suspicious accounts; shelter partnership |
| Scams (fake events) | Stripe payment verification; review system; dispute resolution |
| Abuse in comments | Keyword filtering; community reporting; automatic shadowban for violations |
| Privacy violations | Consent fields; never show full addresses (pin only); GDPR-compliant data retention |

### Trust & Safety Framework
- **Community Guidelines:** Clear rules on acceptable content, consequences for violation
- **Reporting System:** 1-tap report ‚Üí human review within 24h
- **Transparency:** Publish moderation stats monthly (posts removed, accounts banned)
- **Appeals Process:** User can appeal moderation decisions with evidence
